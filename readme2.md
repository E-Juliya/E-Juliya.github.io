# **Инструкция по работе с Git, используя возможности Markdown**
## **Команды Git**
***В терминале можно ввести следующие команды:***
1. Проверить версию Git: 

    * ***git --version***
2. Инициализировать локальный репозиторий:

    * ***git init***
3. Сделать файл отслеживаемым:

    * ***git add ./readme.md***
  
    или сделать все файлы отслеживаемыми:
  
   * ***git add .***  
4. Закоммитить изменения в файле:

    * ***git commit -m "какой-то коммит"***
5. Посмотреть журнал изменений, клавиша Q позволяет выходить из журналов. Head -> - в журнале показывает, в какой ветке сейчас находимся:
    * ***git log***

    или посмотреть более сокращенный вариант журнала изменений:

    * ***git log --oneline***

6. Перейти к нужной сохраненной (закоммиченной) версии внутри ветки:

    * ***git checkout***

    или перейти к последней версии выбранной ветки:
    * ***git checkout master***
7. Сравнить текущий файл с сохраненным файлом (сравниваются только закоммиченные версии):

    * ***git diff***
8. Посмотреть статус папки: 

    * ***git status***
9. Вывести список веток, которые у нас есть (если рядом символ * - в этой ветке сейчас находимся):

    * ***git branch***
10. Создать новую ветку. Создаем в той ветке, от которой нужно создать новую ветку. Проверяем, в какой ветке сейчас находимся командой ***git branch***:

    * ***git branch branch_name***
11. Очистить историю терминала:

    * ***clear***
12. Слить ветки. Сначала нужно перейти в главную ветку, в которую сливаем ***git checkout main_branch_name*** и ввести команду с названием ветки, которую нужно слить с главной:

    * ***git merge alt_branch_name***
13. Удалить ветку (-d - delete):

    * ***git branch -d alt_branch_name***
14. Посмотреть лог коммитов с визуализацией между ними:

    * ***git log --graph***
15. Переименовать текущую ветку:

    * ***git branch -M new_name_branch***
16. Добавить коммит, если файл уже когда-то был отслеживаемым (только если все нужно добавлять):

    * ***git commit -am "какой-то коммит"***
## **Оформление текста при помощи способа форматирования Markdown**
1. Чтобы добавить абзац, нужно оставить между строчками одну пустую строку.
2. Выделить *курсивом* - нужно поставить с двух сторон текста *. 
3. Выделить **полужирным** - для этого нужно поставить ** с каждой стороны текста, который нужно выделить.
4. Выделить ***жирный текст курсивом***  - нужно в начале и в конце выделяемой строки поставить символы ***.
5. Добавить ненумерованный список - для этого перед строкой поставить символ * , - или + .
6. Добавить нумерованный список  - перед строкой проставить 1., 2., 3. и т.д.
7. Добавить заголовок 1-го уровня, используя символ # перед строчкой.
8. Добавить заголовок 2-го уровня, используя символы ## перед строчкой. 
9. Чтобы сделать текст цитируемым, нужно поставить символ < перед цитируемой строкой:
>цитируемый текст
10. Чтобы сделать текст ~~зачеркнутым~~, нужно поставить в начале и в конце текста, который нужно зачеркнуть, символы ~~.
11. Чтобы выделить абзац, нужно 4 раза поставить пробел перед выделяемыми строчками:

        абзац, который нужно выделить
12. Если необходимо выровнять текст по краю (например, заголовки), перед строкой, которая будет выравниваться, нужно поставить столько \&ensp\; , сколько будет достаточно. Поставить перед следующей строчкой  \&ensp\; 8 раз:

&ensp;&ensp;&ensp;&ensp;&ensp;&ensp;&ensp;&ensp;строка с выравниванием

13. Чтобы добавить горизонтальную линию после строки, нужно с новой строки (нажать Enter) ввести символы --- . Получится так:
---
14. Если нужно написать символы, как они есть, которые используются в Markdown, нужно поставить \ перед каждым символом.
15. Чтобы выделить, например, код, нужно в начале и в конце выделяемого текста поставить символ `. Тогда получится:

`code`

16. Чтобы оформить данные в простую таблицу, нужно разделить строки и столбцы символами - и |. Например:

Заголовок1\|Заголовок2

---|---

Значение11|Значение21

Значение12|Значение22 

*Получится:*
Заголовок1|Заголовок2
---|---
Значение11|Значение21
Значение12|Значение22 

## **Пометки**
1. Перед слиянием веток нужно убедиться, что в двух ветках нет конфликтов.
2. Конфликты – когда в разных ветках редактировался один и тот же кусок текста по-разному

+ ***Accept current change*** - *принять текущую версию*

+ ***Accept incoming change*** - *принять изменения с другой ветки*

+ ***Accept both changes*** - *принять оба варианта изменений*

+ ***Compare changes*** - *сравнить изменения*

3. Перед тем, как слить 2 ветки, нужно перейти в ветку, в которую нужно слить (главную) и слить ее с другой веткой.
4. Делать всегда первый коммит после первых базовых настроек и назвать “Initial commit”
5. При создании репозитория создается по умолчанию ветка master(также называется ветка realize). От нее можно создать ветку dev (или pre-realize).
6. Внизу слева пишут ветку (под >Timeline).
7. Команда git log показывает изменения в рамках текущей ветки.

## **Элементы вставки**
1. Вставить картинку: можно добавить в локальный репозиторий картинку, тогда для добавления картинки в файл нужно прописать следующее: ![текст](полный путь картинки в папке репозитория/имя файла.расширение файла "необязательный текст при наведении на картинку"). Например, ![тут котики] (kats.jpg "котики"):
![тут котики](kats.jpg "котики")
2. Вставить ссылку можно так: \[Справочник по Docs Markdown\]\(https://learn.microsoft.com/ru-ru/contribute/markdown-reference\). Получится: [Справочник по Docs Markdown](https://learn.microsoft.com/ru-ru/contribute/markdown-reference) 
3. Вставить смайлики. Сделать такую запись: ***:smiley:*** или использовать таблицу символов (при помощи Alt-кодов), например: 

Alt +1: ☺ 

Alt + 14: ♫ 

Alt + 15: ☼
## **Игнорирование файлов**
1. Создаем файл в папке репозитория с названием .gitignore.
2. Открываем сам файл редактирования и вводим название.расширение столько файлов, сколько нужно будет игнорировать Git.
## **Работа с удаленными репозиториями"**
***Сервис GitHub*** (https://github.com/) – сервис MS которая позволяет интегрироваться с программой Git и настроить удаленную работу.

Указать удаленный репозитория для локального репозитория (достаточно одного раза): 

git remote add origin ссылка

Указать, какая ветка будет являться основной:

* ***git branch -M название ветки***

Отправить (направить, толкать) то, что есть в локальном репозитории, в основную ветку в удаленном репозитории:

* ***git push -u origin название ветки***

Все изменения, которые будут проведены в локальном репозитории не отразятся в удаленном репозитории. Для того, чтобы в привязанном удаленном репозитории отобразились изменения, которые производились в локальном репозитории, нужно ввести:

* ***git push*** 

Забрать (тянуть) с удаленного репозитория в локальный репозиторий (не только загружает изменения, но и мержит ветки):

* ***git pull***

**Fork** – сделать полную отдельную копию репозитория.

## **Сохранение изменений**
Чтобы сохранить изменения, в терминале ввести команды:

`git add .`

`git commit -m "коммит"`

И сохранить файл комбинацией Ctrl + S или File - Save
